# Tradeoff Analysis Protocol

1. Always identify the options being compared
2. List pros and cons for each option
3. Consider performance, maintainability, scalability, and team expertise
4. Document the decision rationale

## When to Use This Skill

- Multiple valid approaches exist for a problem
- Choosing between technologies or frameworks
- Deciding on architecture patterns
- Evaluating performance vs. development speed tradeoffs

## Process

1. **Identify the decision points** that need analysis
2. **List all viable options** with their characteristics
3. **Analyze each option** across relevant dimensions:
   - Performance implications
   - Development time
   - Maintenance overhead
   - Learning curve
   - Scalability
   - Security considerations
4. **Document the recommendation** with clear rationale
5. **Note alternatives considered** for future reference
