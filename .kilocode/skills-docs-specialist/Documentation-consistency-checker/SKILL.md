# Documentation Consistency Check Protocol

1. Always verify alignment with actual implementation
2. Check for outdated information
3. Ensure consistent terminology
4. Validate completeness of information

## When to Use This Skill

- Reviewing existing documentation
- Updating documentation after changes
- Ensuring documentation quality
- Preparing documentation for release

## Process

1. **Compare with implementation** - Verify accuracy
2. **Check terminology** - Consistent naming and terms
3. **Validate examples** - Working and relevant
4. **Assess completeness** - All features documented
5. **Review formatting** - Consistent style
6. **Verify links** - All references work
7. **Check versioning** - Correct version information
8. **Ensure accessibility** - Clear and understandable
