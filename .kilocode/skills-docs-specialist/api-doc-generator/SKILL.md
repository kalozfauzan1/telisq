# API Documentation Generation Protocol

1. Always extract interface definitions accurately
2. Document request/response formats
3. Include example usage
4. Specify error conditions

## When to Use This Skill

- Creating API documentation
- Updating existing API specs
- Generating SDK documentation
- Creating user guides for APIs

## Process

1. **Extract endpoints** - All available routes
2. **Document parameters** - Path, query, body parameters
3. **Specify request format** - Headers, payload structure
4. **Detail response format** - Success and error responses
5. **Include examples** - Sample requests and responses
6. **Define authentication** - Required headers/tokens
7. **Document rate limits** - Throttling information
8. **Version appropriately** - API versioning scheme
