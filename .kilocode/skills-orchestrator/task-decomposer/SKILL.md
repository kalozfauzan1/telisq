# Task Decomposition Protocol

1. Always identify the main objective
2. Break into logical, sequential steps
3. Estimate effort and dependencies
4. Create clear completion criteria

## When to Use This Skill

- Tackling large, complex projects
- Planning multi-step implementations
- Coordinating work across different areas
- Managing complex refactoring efforts

## Process

1. **Define the main objective** - Clear end goal
2. **Identify major components** - High-level breakdown
3. **Break into specific tasks** - Actionable steps
4. **Estimate effort** - Time and complexity assessment
5. **Identify dependencies** - Task relationships
6. **Define success criteria** - Completion conditions
7. **Prioritize tasks** - Order of execution
