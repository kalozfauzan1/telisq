# Dependency Mapping Protocol

1. Always identify all system components
2. Map both direct and indirect dependencies
3. Distinguish between different types of dependencies
4. Document impact of changes

## When to Use This Skill

- Planning system changes or refactors
- Understanding system architecture
- Identifying coupling and cohesion issues
- Planning modular development

## Process

1. **Inventory components** - List all system parts
2. **Map direct dependencies** - Explicit connections
3. **Identify implicit dependencies** - Hidden connections
4. **Categorize dependency types** - Runtime, compile-time, data, etc.
5. **Assess coupling strength** - Tight vs loose coupling
6. **Document impact chains** - How changes propagate
7. **Recommend improvements** - Reduce unnecessary dependencies
