# Architecture Diagram Generation Protocol

1. Always identify the scope of the diagram needed
2. Use appropriate diagram type (component, sequence, deployment, etc.)
3. Include relevant system boundaries and interactions
4. Keep diagrams clear and readable

## When to Use This Skill

- Designing system architecture
- Documenting existing systems
- Planning microservices interactions
- Visualizing data flows

## Process

1. **Identify the purpose** of the diagram
2. **Choose appropriate diagram type**:
   - Component diagrams for system structure
   - Sequence diagrams for interactions
   - Deployment diagrams for infrastructure
   - Flow charts for processes
3. **Define system boundaries** and external dependencies
4. **Document relationships** and data flows
5. **Generate Mermaid or other format** diagram
6. **Explain key elements** and their purposes
