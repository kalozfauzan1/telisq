# Architectural Tradeoff Documentation Protocol

1. Always document the context and requirements
2. List all considered alternatives
3. Record decision factors and rationale
4. Note consequences and future considerations

## When to Use This Skill

- Making significant architectural decisions
- Recording technology choices
- Documenting design decisions
- Creating architecture decision records (ADRs)

## Process

1. **Document the context** - Problem being solved
2. **List the requirements** - Constraints and objectives
3. **Identify alternatives** - Options considered
4. **Analyze tradeoffs** - Pros and cons of each
5. **Record decision** - Chosen solution and rationale
6. **Note consequences** - Implications and future considerations
7. **Update regularly** - As context changes
