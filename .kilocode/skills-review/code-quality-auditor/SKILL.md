# Code Quality Audit Protocol

1. Always assess code against established standards
2. Identify maintainability issues
3. Check for security vulnerabilities
4. Review performance implications

## When to Use This Skill

- Performing code reviews
- Assessing legacy code quality
- Establishing coding standards
- Preparing for refactoring efforts

## Process

1. **Review structure** - Organization and modularity
2. **Check naming conventions** - Clarity and consistency
3. **Analyze complexity** - Cyclomatic and cognitive complexity
4. **Verify error handling** - Proper exception management
5. **Assess test coverage** - Adequacy of tests
6. **Evaluate performance** - Efficiency considerations
7. **Identify security issues** - Potential vulnerabilities
8. **Document findings** - Clear recommendations
