# Test Coverage Analysis Protocol

1. Always measure both quantity and quality of tests
2. Identify critical paths with insufficient coverage
3. Assess test effectiveness
4. Recommend improvements

## When to Use This Skill

- Evaluating test suite effectiveness
- Planning testing strategy
- Assessing risk in untested code
- Improving existing test coverage

## Process

1. **Measure coverage metrics** - Lines, branches, functions
2. **Identify uncovered areas** - Critical vs non-critical
3. **Assess test quality** - Meaningful vs superficial tests
4. **Analyze test types** - Unit, integration, end-to-end
5. **Evaluate test maintenance** - Fragility and reliability
6. **Recommend priorities** - Focus on high-risk areas
7. **Suggest improvements** - Better testing approaches
