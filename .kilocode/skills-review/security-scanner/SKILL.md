# Security Scanning Protocol

1. Always identify potential vulnerability types
2. Check for common security flaws
3. Review configuration settings
4. Assess data handling practices

## When to Use This Skill

- Performing security reviews
- Assessing application security posture
- Preparing for penetration testing
- Reviewing third-party integrations

## Process

1. **Identify vulnerability types** - Injection, XSS, CSRF, etc.
2. **Scan input validation** - Sanitization and validation
3. **Review authentication** - Secure implementation
4. **Check authorization** - Proper access controls
5. **Assess data protection** - Encryption and storage
6. **Evaluate dependencies** - Vulnerable packages
7. **Review configurations** - Secure settings
8. **Document findings** - Risk levels and remediation
